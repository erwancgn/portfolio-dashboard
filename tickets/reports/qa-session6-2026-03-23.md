## Vérifications spécifiques — API Routes

- Route dans `src/app/api/*` : PASS — `src/app/api/positions/route.ts`
- Clé secrète exposée côté client : PASS (aucune dans `route.ts`)
- `NEXT_PUBLIC_APP_URL` dans `PositionsTable.tsx` : NOTE — `process.env.NEXT_PUBLIC_APP_URL` est utilisé ligne 67. Ce n'est pas une clé secrète (c'est une URL publique), donc non une violation de sécurité. Cependant, dans un Server Component, `process.env.APP_URL` (sans préfixe `NEXT_PUBLIC_`) serait plus approprié car cette variable n'a pas besoin d'être exposée au client.

# Checklist — Story Draft

Avant d'envoyer une story au dev-agent, vérifier :

- [ ] La story a un titre clair et un identifiant (US-XXX)
- [ ] Le contexte explique le lien avec l'epic/PRD
- [ ] Les actions sont listées avec les fichiers concernés
- [ ] Les critères d'acceptation sont binaires et testables
- [ ] Le hors périmètre est défini
- [ ] Les notes techniques incluent les types DB / API pertinents
- [ ] La story a une section `## Estimation` avec taille (XS/S/M/L/XL) et justification
- [ ] La taille détermine la route : XS=direct · S/M=dev-agent · L=architect si DB · XL=BMAD
- [ ] La story est implémentable en une session (~1h max)
- [ ] Aucune décision d'architecture implicite dans la story

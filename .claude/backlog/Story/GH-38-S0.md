# GH-38-S0 — [Migration DB] Colonne soft delete sur positions

## Contexte
Prérequis bloquant de l'EPIC 12 (GH-38) Corbeille & suppression douce. Sans cette migration, aucune des stories US-COR-01 et US-COR-02 ne peut être implémentée. La colonne `deleted_at` est le marqueur central du mécanisme de suppression douce.

## Description
En tant que dev-agent, je veux ajouter la colonne `deleted_at` sur la table `positions` et adapter les RLS policies, afin que le soft delete soit possible sans casser les requêtes existantes.

## Ce qui doit être fait
- [ ] Créer la migration SQL `20260403_add_soft_delete_to_positions.sql` avec la colonne `deleted_at TIMESTAMPTZ DEFAULT NULL`
- [ ] Ajouter un index partiel `WHERE deleted_at IS NULL` pour optimiser les lectures du portfolio
- [ ] Mettre à jour la RLS policy `SELECT` existante pour filtrer `deleted_at IS NULL` (positions actives)
- [ ] Créer une RLS policy distincte permettant de lire les lignes `deleted_at IS NOT NULL` (pour la corbeille)
- [ ] Régénérer `src/types/database.ts` via `npx supabase gen types typescript --local`
- [ ] Vérifier que les positions existantes ont bien `deleted_at = NULL` après migration

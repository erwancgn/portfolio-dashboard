# GH-22 — [US-011] Configurer une règle DCA

## Contexte
Partie de l'EPIC 4 DCA (GH-4). Permet au PO de définir un montant et une fréquence de DCA pour chaque titre, afin d'automatiser le suivi de sa stratégie d'investissement régulier. Reporté en v1.5.

## Description
En tant que PO, je veux définir un montant et une fréquence de DCA pour chaque titre, afin d'automatiser le suivi de ma stratégie d'investissement régulier.

## Ce qui doit être fait
- [ ] Créer la migration DB pour la table `dca_rules` (montant, fréquence, actif, next_expected_at)
- [ ] Créer/étendre `src/app/api/dca/route.ts` — GET liste des règles, POST/PUT save règle
- [ ] Créer le composant `src/components/dca/DcaRulesTable.tsx` — tableau avec toggle actif/inactif
- [ ] Afficher la date du prochain DCA et l'indicateur de retard par ligne
- [ ] Sauvegarde immédiate en base au changement

## Fichiers concernés
| Fichier | Action |
|---------|--------|
| `supabase/migrations/` | créer — migration table `dca_rules` |
| `src/app/api/dca/route.ts` | modifier — ajouter GET list + PUT update règle |
| `src/components/dca/DcaRulesTable.tsx` | créer — tableau de configuration DCA |
| `src/types/database.ts` | régénérer après migration |

## Critères d'acceptation
- [ ] CA1 : Tableau listant toutes les positions avec colonne montant DCA et fréquence
- [ ] CA2 : Toggle actif/inactif par ligne, changement sauvegardé immédiatement en base
- [ ] CA3 : Date du prochain DCA affichée et calculée correctement pour chaque fréquence (hebdo, mensuel, trimestriel)
- [ ] CA4 : Indicateur visuel si le DCA est en retard (date dépassée)
- [ ] CA5 : Les règles survivent à un rechargement de page

## Hors périmètre
- L'enregistrement effectif d'un passage DCA (GH-23)
- L'historique des passages (GH-24)
- Les notifications ou alertes

## Notes techniques
- Table `dca_rules` : `id, user_id, position_id, amount_eur, frequency (weekly|monthly|quarterly), is_active, next_expected_at, created_at`
- Calcul `next_expected_at` : date du dernier passage + fréquence, ou date de création si aucun passage
- `src/app/api/dca/route.ts` existe déjà — vérifier son contenu avant modification
- Utiliser `src/lib/supabase/server.ts` dans la route API

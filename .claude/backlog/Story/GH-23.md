# GH-23 — [US-012] Enregistrer un passage DCA

## Contexte
Partie de l'EPIC 4 DCA (GH-4). Permet au PO d'enregistrer un achat DCA effectué et de mettre à jour automatiquement le PRU et la quantité de la position. Reporté en v1.5.

## Description
En tant que PO, je veux cliquer sur "DCA effectué" pour enregistrer mes achats réguliers, afin de mettre à jour automatiquement mon PRU et mes positions.

## Ce qui doit être fait
- [ ] Créer la migration DB pour la table `dca_history` (position_id, amount, price, quantity, new_pru, executed_at)
- [ ] Créer la route `POST /api/dca/execute` — calcul PRU, insert history, update position
- [ ] Créer le composant `src/components/dca/DcaExecuteModal.tsx` — pop-up avec pré-remplissage
- [ ] Implémenter le recalcul PRU moyen pondéré
- [ ] Recalculer `next_expected_at` sur la règle DCA correspondante

## Fichiers concernés
| Fichier | Action |
|---------|--------|
| `supabase/migrations/` | créer — migration table `dca_history` |
| `src/app/api/dca/route.ts` | modifier — ajouter POST /execute |
| `src/components/dca/DcaExecuteModal.tsx` | créer — modal d'enregistrement passage |
| `src/app/api/positions/[id]/route.ts` | modifier — mise à jour PRU et quantité |
| `src/types/database.ts` | régénérer après migration |

## Critères d'acceptation
- [ ] CA1 : Pop-up affiche uniquement les titres avec une règle DCA active
- [ ] CA2 : Montant pré-rempli depuis la règle DCA, prix d'exécution à saisir manuellement
- [ ] CA3 : Quantité calculée automatiquement = montant / prix saisi
- [ ] CA4 : Confirmation recalcule le PRU selon la formule `(ancienne_qté × ancien_pru + montant) / (ancienne_qté + qté_achetée)`
- [ ] CA5 : `next_expected_at` de la règle DCA est recalculé après enregistrement
- [ ] CA6 : Message de succès avec récapitulatif (ticker, qté, nouveau PRU)

## Hors périmètre
- La configuration des règles DCA (GH-22)
- La visualisation de l'historique (GH-24)
- Les notifications push ou email

## Notes techniques
- Formule PRU : `nouveau_pru = (ancienne_quantité × ancien_pru + montant) / (ancienne_quantité + quantité_achetée)`
- La quantité achetée = `amount_eur / execution_price`
- Utiliser une transaction Supabase pour l'insert history + update position (atomicité)
- Bloquée par GH-22 (table `dca_rules` doit exister)

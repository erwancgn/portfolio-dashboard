# GH-87 — [US-UX-06] Refonte encart changelog / versions

## Contexte
Le panneau affichant l'historique des versions est illisible : texte dense, pas de hiérarchie visuelle, badges de version peu visibles, pas d'espacement entre les entrées.

## Description
En tant qu'utilisateur, je veux voir un changelog clair et lisible, inspiré du style minimaliste Trade Republic.

## Ce qui doit être fait
- [ ] Localiser le composant changelog/version (ChangelogPanel, VersionHistory, ou similaire)
- [ ] Refondre le layout :
  - Badge version coloré (fond vert/gris selon majeur/mineur) à gauche
  - Titre bold à côté
  - Date alignée à droite, couleur secondaire
  - Bullets avec meilleur espacement
  - Séparateur entre versions
- [ ] Couleurs et typographie cohérentes avec le thème (CSS variables `var(--color-*)`)
- [ ] Pas de `shadow-lg`, `border-2+`, couleurs vives

## Critères d'acceptation
- [ ] CA1 : Chaque version a un badge, un titre bold, une date lisible
- [ ] CA2 : Les bullets sont lisibles avec espacement suffisant
- [ ] CA3 : Le style est cohérent avec le reste de l'app (thème light)
- [ ] CA4 : Pas de régression sur le contenu affiché

## Estimation
S — refonte visuelle d'un composant existant

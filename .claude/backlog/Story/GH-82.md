# GH-82 — [US-UX-01] Supprimer le bouton "← Retour" du header

## Contexte
Le header des pages secondaires (Analyse, Chat, etc.) affiche un bouton "← Retour" qui crée un comportement non-naturel. La navigation doit être assurée par les onglets/liens du header, pas par un bouton retour.

## Description
En tant qu'utilisateur, je veux naviguer entre les pages de façon fluide via le menu principal, sans bouton "Retour" qui casse l'expérience.

## Ce qui doit être fait
- [ ] Localiser le composant header qui affiche le bouton "← Retour"
- [ ] Supprimer le bouton retour
- [ ] Vérifier que la navigation reste cohérente sur toutes les pages secondaires
- [ ] S'assurer qu'on peut revenir au dashboard via le menu ou le logo

## Critères d'acceptation
- [ ] CA1 : Aucun bouton "← Retour" visible sur aucune page
- [ ] CA2 : Navigation possible vers le dashboard depuis toutes les pages secondaires
- [ ] CA3 : Pas de régression sur le layout du header

## Estimation
XS — suppression ciblée d'un élément

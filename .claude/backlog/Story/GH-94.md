# GH-94 — [US-IA-05] Agent Haiku 4.5 pour tâches triviales

## Contexte

Le protocol `agent-protocol.md` définit 3 tiers de modèles :
- Haiku 4.5 : tâches simples (1-2 fichiers, bug évident)
- Sonnet 4.6 : tâches standard (défaut)
- Opus 4.6 : tâches complexes

En pratique, Haiku 4.5 n'est **jamais utilisé** — tout passe par Sonnet (10x plus cher). Sur les sessions de debug simple, on paie Sonnet pour des corrections triviales.

## Description

En tant que PO, je veux que les bugs simples (typo, 1-ligne, config) soient traités par Haiku 4.5 automatiquement, sans avoir à le spécifier manuellement.

## Ce qui doit être fait

- [ ] Créer `.claude/agents/haiku-dev.md` : agent Haiku 4.5 avec outils Read, Edit, Bash (tsc, eslint), Glob, Grep — pas de Write, pas de WebSearch
- [ ] Mettre à jour `agent-protocol.md` : ajouter les critères de déclenchement explicites du tier Haiku :
  - modif 1 fichier, < 10 lignes changées
  - correction bug dont la cause est identifiée dans le message
  - renommage, typo, ajout entrée config
- [ ] Mettre à jour le skill `dev-workflow` pour router vers `haiku-dev` si la tâche matche les critères
- [ ] Ajouter dans LESSONS.md les 3 patterns qui déclenchent Haiku vs Sonnet

## Critères d'acceptation

- [ ] CA1 : Un bug 1-ligne est traité par `haiku-dev`, pas `dev-agent` (Sonnet)
- [ ] CA2 : `haiku-dev` ne peut pas créer de nouveaux fichiers (pas d'outil Write)
- [ ] CA3 : Si Haiku échoue ou si la tâche s'avère plus complexe, escalade vers `dev-agent` (Sonnet)
- [ ] CA4 : `agent-protocol.md` documente clairement le seuil de routage

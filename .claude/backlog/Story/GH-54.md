# GH-54 — [US-020] Export CSV et PDF du portfolio

## Contexte
Permet au PO d'exporter son portfolio dans un format exploitable hors de l'app (tableur, archive, comptable).

## Description
En tant que PO, je veux exporter mon portfolio en CSV ou PDF, afin de l'analyser dans un tableur ou de l'archiver.

## Ce qui doit être fait
- [ ] Créer la route `GET /api/export/csv` — génération CSV des positions
- [ ] Ajouter le bouton export CSV dans le composant de la page dashboard
- [ ] (Optionnel) Export PDF — version imprimable du dashboard

## Fichiers concernés
| Fichier | Action |
|---------|--------|
| `src/app/api/export/csv/route.ts` | créer — génération CSV |
| `src/app/dashboard/page.tsx` | modifier — ajouter bouton export |

## Critères d'acceptation
- [ ] CA1 : L'export CSV contient les colonnes : ticker, nom, type, quantité, PRU, prix actuel, valeur, P&L€, P&L%
- [ ] CA2 : Le fichier CSV est téléchargé avec le nom `portfolio_YYYY-MM-DD.csv`
- [ ] CA3 : Le CSV est encodé en UTF-8 avec séparateur `;` (compatibilité Excel FR)
- [ ] CA4 : L'export fonctionne même avec un portfolio de 50+ positions

## Hors périmètre
- Export PDF (DOD optionnel — hors périmètre MVP de cette story)
- Export de l'historique des transactions
- Export des données DCA

## Notes techniques
- Pas de dépendance externe requise — génération CSV native en string JS
- Utiliser `Content-Disposition: attachment; filename=...` dans les headers HTTP
- Prix actuels : utiliser le dernier prix connu en base (pas d'appel API quote au moment de l'export)

## Dépendances
- Bloquée par : aucune
- Débloque : rien

## Estimation
S — 1 route API + 1 modification composant

# GH-86 — [US-UX-05] Formulaire position — auto-fill à la sélection d'une suggestion

## Contexte
Quand l'utilisateur tape un ticker ou un nom dans le formulaire "Nouvelle position" et sélectionne une suggestion FMP, seul le champ saisi est rempli. Les champs ISIN, Secteur, Nom et Ticker restent vides et doivent être remplis manuellement.

## Description
En tant qu'utilisateur, je veux que la sélection d'une suggestion remplisse automatiquement tous les champs disponibles (Ticker, Nom, ISIN, Secteur, Pays) sans que j'aie à les saisir un par un.

## Ce qui doit être fait
- [ ] Lire le composant formulaire d'ajout de position (AddPositionModal ou équivalent)
- [ ] À la sélection d'une suggestion : remplir Ticker, Nom, ISIN, Secteur, Pays depuis la réponse FMP
- [ ] Vérifier que les données FMP retournées contiennent bien ces champs (profil ou search endpoint)
- [ ] Si un champ n'est pas disponible dans la réponse, le laisser vide (pas d'erreur)
- [ ] Fermer le dropdown après sélection (lié à GH-85)

## Critères d'acceptation
- [ ] CA1 : Sélectionner une suggestion remplit automatiquement Ticker + Nom
- [ ] CA2 : Si disponible dans la réponse FMP : ISIN, Secteur, Pays également remplis
- [ ] CA3 : L'utilisateur peut modifier les champs pré-remplis avant de soumettre
- [ ] CA4 : Aucune régression sur la soumission du formulaire

## Notes techniques
- L'endpoint FMP search (`/search?query=`) retourne `symbol`, `name`, `exchangeShortName`
- L'endpoint FMP profile (`/profile/{symbol}`) retourne `isin`, `sector`, `country`
- Vérifier si un appel profile est déclenché à la sélection ou si les données sont déjà dans le search

## Estimation
M — modification formulaire + potentiellement un appel API supplémentaire au clic

# GH-27 — [TASK-009] PWA (Progressive Web App)

## Contexte
Partie de l'EPIC 5 Infrastructure (GH-5). Permet d'installer l'app sur l'écran d'accueil d'un iPhone et de l'ouvrir en plein écran comme une app native.

## Description
En tant que PO, je veux installer l'app sur mon écran d'accueil iPhone, afin d'y accéder rapidement comme une app native.

## Ce qui doit être fait
- [ ] Installer le package `next-pwa`
- [ ] Créer `public/manifest.json` avec les métadonnées de l'app
- [ ] Modifier `next.config.js` (ou `next.config.ts`) pour activer next-pwa
- [ ] Créer les icônes PWA (192x192 et 512x512) dans `public/icons/`
- [ ] Ajouter les meta tags PWA dans `src/app/layout.tsx`

## Fichiers concernés
| Fichier | Action |
|---------|--------|
| `public/manifest.json` | créer — manifeste PWA |
| `public/icons/` | créer — icônes 192px et 512px |
| `next.config.js` / `next.config.ts` | modifier — configurer next-pwa |
| `src/app/layout.tsx` | modifier — meta tags PWA |
| `package.json` | modifier — ajouter next-pwa |

## Critères d'acceptation
- [ ] CA1 : Safari sur iPhone affiche le bouton "Ajouter à l'écran d'accueil"
- [ ] CA2 : L'app s'ouvre en plein écran (sans barre Safari) depuis l'écran d'accueil
- [ ] CA3 : L'icône de l'app apparaît correctement sur l'écran d'accueil iPhone
- [ ] CA4 : Le build Next.js reste fonctionnel après l'ajout de next-pwa

## Hors périmètre
- Le mode hors ligne (Service Worker cache)
- Les notifications push
- Le support Android / autres navigateurs

- next-pwa peut nécessiter une configuration spécifique pour Next.js 16 (App Router)
- Les icônes peuvent être générées depuis une image source avec un outil CLI

# GH-38 — [EPIC 12] Corbeille & suppression douce

## Objectif
Permettre au PO de restaurer une position supprimée par erreur, en implémentant une suppression douce (soft delete) sur les positions.

## User Stories
- [ ] US-COR-01 : Soft delete d'une position (à créer)
- [ ] US-COR-02 : Vue corbeille et restauration (à créer)

## Dépendances
- Migration DB requise : colonne `deleted_at` sur table `positions`
- Story de migration DB à créer en premier

## Critères d'acceptation (epic-level)
- [ ] Supprimer une position la déplace en corbeille, ne l'efface pas
- [ ] La corbeille est accessible depuis l'interface
- [ ] Une position en corbeille peut être restaurée
- [ ] La corbeille peut être vidée définitivement

# GH-38 — [EPIC 12] Corbeille & suppression douce

## Objectif
Permettre au PO de restaurer une position supprimée par erreur, en implémentant une suppression douce (soft delete) sur les positions.

## User Stories
- [ ] [GH-38-S0](../Story/GH-38-S0.md) — Migration DB : colonne `deleted_at` sur `positions` *(prérequis bloquant)*
- [ ] [GH-38-S1](../Story/GH-38-S1.md) — US-COR-01 : Soft delete d'une position *(bloquée par S0)*
- [ ] [GH-38-S2](../Story/GH-38-S2.md) — US-COR-02 : Vue corbeille et restauration *(bloquée par S0 + S1)*

## Ordre d'implémentation
```
GH-38-S0 (Migration DB)
    └── GH-38-S1 (Soft delete API)
            └── GH-38-S2 (Vue corbeille)
```

## Dépendances
- Migration DB requise : colonne `deleted_at` sur table `positions` (GH-38-S0)

## Critères d'acceptation (epic-level)
- [ ] Supprimer une position la déplace en corbeille, ne l'efface pas
- [ ] La corbeille est accessible depuis l'interface
- [ ] Une position en corbeille peut être restaurée
- [ ] La corbeille peut être vidée définitivement

# GH-89 — [EPIC] Améliorations IA backend

## Objectif

Réduire les coûts LLM et améliorer la performance du backend IA (routes `/api/analyse/*`) ainsi que l'efficacité du workflow de développement IA (agents Claude Code).

## Périmètre

### Production IA (`/api/analyse/*`)
- Unifier le client LLM (Gemini/Groq) en service central
- Remplacer l'extraction JSON par regex par le mode structuré Gemini natif
- Mettre en cache les données FMP (données fondamentales, pas les cours)
- Déplacer le quota tokens côté serveur

### Dev IA (Claude Code agents)
- Activer le tier Haiku 4.5 pour les tâches triviales
- Court-circuiter le pipeline BMAD pour les tickets déjà définis
